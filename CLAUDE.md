# 毕业论文 RAG 知识问答系统 — 技术全貌

## 项目定位
让大学生用自然语言询问毕业论文相关政策，系统基于学校官方 PDF/Word 文档给出准确回答。

## 检索架构（完整链路）
```
用户提问 → 缓存检查 → 查询扩展(LLM改写3-4个搜索词) → 双路粗排
→ RRF融合(top-20) → Cross-Encoder精排(top-5) → LLM流式生成 → 缓存写入
```

## 双路粗排
- **向量检索**: BAAI/bge-small-zh-v1.5 → ChromaDB 余弦相似度 (本地免费)
- **BM25检索**: jieba分词 → rank_bm25 关键词匹配 (本地免费)
- **融合**: RRF (Reciprocal Rank Fusion, k=60)

## 精排
- **模型**: BAAI/bge-reranker-v2-m3 Cross-Encoder (本地免费)
- **作用**: 对粗排top-20候选逐个打分，挑出最相关的top-5

## LLM配置
- **默认**: DeepSeek API (deepseek-chat)
- **备选**: Claude API / OpenAI / 硅基流动

## Embedding配置
- **默认**: 本地 BGE 模型 (免费离线)
- **备选**: Voyage AI / 硅基流动

## 文档处理
- PDF: PyMuPDF 提取电子文本 + 表格识别
  → 无文字时自动 OCR (PaddleOCR, 中文准确率 ~98%, pip 一步安装)
- Word: python-docx，段落+表格提取
- ZIP: 自动解压递归处理
- 分块: 智能识别标题/章节边界，800字符/块，150字符重叠，表格独立分块

## 反幻觉机制
- System Prompt 严格约束: 只使用文档明确内容，禁止推理联想
- 低相关片段标注，LLM被告知不可作为答案依据
- Cross-Encoder精排过滤不相关内容

## UI
- Streamlit 三页面: 文档导入 / 知识问答 / 文档管理
- 本地文件夹导入: 输入路径直接扫描，无文件大小限制
- 流式回答 + 引用来源展示 + 匹配类型标注

## 缓存策略
- 语义匹配: 用问题 Embedding 余弦相似度 (≥0.92) 匹配，非精确文本匹配
  → "论文格式要求" 和 "论文的格式有什么规定" 命中同一缓存
- LRU 淘汰: 按最近访问时间淘汰，高频问题被保护
- 置信度过滤: Reranker 最高分 < 0.5 不缓存，"未找到" 回答不缓存
- 上限 500 条，文档变动(导入/删除)自动清空

## 设计决策记录
1. 选择本地BGE而非API Embedding → 节省成本
2. 添加BM25补充向量检索 → 解决"单面打印"搜不到"装订要求"
3. 添加查询扩展 → 多角度改写问题，提升召回
4. 添加Cross-Encoder精排 → 粗排多取候选，精排过滤噪声
5. 强制System Prompt约束 → 防止LLM从"装订要求"推断出"单面打印"
6. 不走微调 → RAG更适合知识库型问答，知识更新只需重新上传文档
